So What

The model is tuned to over-flag rather than miss — a clinician still makes every diagnosis, but the queue could surface likely-malignant patches first, cutting time-to-review on the cases that matter most.

01 — Diagnostic Support

Triage, not autonomous diagnosis

Flags high-risk patches for prioritized review — a clinician makes every call, not the model.

02 — Recall-Optimized Design

Tuned for the cost asymmetry in oncology

A missed malignant case costs far more than a false alarm, so the threshold is deliberately biased toward sensitivity.

03 — Patient-Level Validation

Evaluated on genuinely unseen patients

Train/validation/test splits were made at the patient level, so reported performance isn't inflated by data leakage.

Methodology: EfficientNetB0 transfer learning (TensorFlow/Keras), patient-level split, IDC histopathology dataset, 224×224px patches — recall-optimized threshold (0.449) Educational / Portfolio Prototype — Not For Clinical Use
